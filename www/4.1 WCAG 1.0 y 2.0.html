4.1 WCAG 1.0 y 2.0

La accesibilidad web tiene como objetivo lograr que las páginas web sean utilizables por el máximo número de
personas en cualquier equipo utilizado, independientemente de sus conocimientos o capacidades personales.

Para lograr la accesibilidad, se han elaborado diferentes pautas que explican cómo se tienen que crear las
páginas web para que sean accesibles.

Las pautas que vamos a tratar en este manual son WCAG 1.0 y WCAG 2.0.

WCAG 1.0

WCAG (Web Content Accessibility Guidelines) 1.0 es una recomendación elaborada el 5 de Mayo de 1999 por
W3C que explica cómo hacer el contenido web accesible a las personas con discapacidad.

Se compone de 14 pautas, que son los principios generales para hacer diseño accesible:

  1. La página web debe proporcionar alternativas equivalentes para el contenido visual y auditivo, para beneficiar a personas ciegas y/o sordas, o a aquellas con limitada conexión a internet.

  2. La web no se base sólo en el color, comprendiéndose todo el texto sin necesidad de verlo con colores.

  3. Se tienen que utilizar marcadores y hojas de estilo apropiadamente, en vez de utilizar atributos de presentación.

  4. Se debe especificar el idioma usado.

  5. Las tablas se tienen que transformar correctamente

  6. Los contenidos de las páginas que usen nuevas tecnologías deben transformarse correctamente, para poder ser visualizados por navegadores antiguos.

  7. El usuario debe controlar los cambios de los contenidos tempo-dependientes, para facilitar la lectura a personas que no pueden leer contenidos en movimiento.

  8. Debe asegurar la accesibilidad directa de las interfaces que se incrusten en la página, o lo que es lo mismo, que un objeto incrustado (flash, applet,..) debe ser accesible.

  9. La página debe ser diseñada para ser utilizada independientemente del dispositivo que se utilice, ya sea usando un ratón, teclado, etc.

  10. Utilizar soluciones provisionales, hasta que antiguos navegadores operen correctamente.

  11. Utilizar las tecnologías de W3C, ya que incorporan características accesibles y se pueden usar como base.

  12. Proporcionar información de contexto, y orientación, para comprender elementos complejos, y guiar a personas con discapacidad cognitiva y visual.

  13. Proporcionar mecanismos para facilitar la búsqueda de información, tales como barras de navegación, mapas web, etc.

  14. Utilizar lenguaje claro y simple para los documentos.

WCAG 2.0

WCAG 1.0 fue reemplazado por WCAG 2.0 en diciembre de 2008, que a su vez ha sido reemplazado por WCAG 2.1 en junio de 2018.

La mayoría de los sitios web que se ajusten a WCAG 1.0 no debería requerir cambios significativos a fin de ajustarse a WCAG 2.0.

WCAG 2.0 nace para suplir a su versión predecesora, ya que la 1.0 fue publicada en 1999, y se ha quedado un poco obsoleta con respecto a las nuevas tecnologías.

  • WCAG2.0 se organiza según estos niveles:

    • 4 Principios:

    • Perceptible: la información y los componentes de la interfaz de usuario deben ser mostrados a los usuarios en formas que ellos puedan percibir.
    • Operable: los componentes de la interfaz de usuario y la navegación debe ser manejable.
    • Comprensible: la información y la utilización del interfaz de usuario deben ser comprensibles.
    • Robusto: el contenido deber ser suficientemente robusto para que pueda ser bien interpretado por una gran variedad de agentes de usuario, incluyendo los productos de apoyo.

    12 Directrices (guidelines):

    Cada principio se divide en diferentes directrices:
      1. Perceptible: 4 directrices
      • Texto alternativo.
      • Contenido multimedia dependiente del tiempo
      • Adaptable
      • Distinguible

      2. Operable: 4 directrices
      • Teclado accesible
      • Tiempo suficiente
      • Ataques epilépticos
      • Navegación

      3. Comprensible: 3 directrices
      • Legible
      • Previsible
      • Asistencia a la entrada de datos

      4. Robusto: 1 directriz
      • Compatible

    60 Criterios de cumplimiento (o conformidad):

    • Cada directriz se compone de criterios de cumplimiento (éxito) que describen específicamente que se debe de cumplir para lograr la accesibilidad
    • Cada criterio de cumplimiento está escrito como una exposición que será verdadera o falsa cuando el contenido web específico sea probado contra él
    • Con el fin de satisfacer las necesidades de diferentes grupos de usuarios y de diferentes situaciones, cada criterio de cumplimiento tiene asignado un nivel de conformidad (adecuación): A, AA y AAA Éste apartado lo estudiaremos con más detalle en la siguiente sección del manual (4.2 Criterios de coformidad de WCAG 2.0)

    Técnicas suficientes:

    • Son Pruebas específicas para comprobar si se cumplen los criterios de cumplimiento
    • Son numerosas técnicas, numeradas desde H2, H4, H24, H25... hasta H94.
    • WCAG Working Group considera que son suficientes para lograr el criterio de cumplimiento
    • Aunque cierto contenido cumpla el nivel de conformidad AAA, puede ser que no sea accesible para usuarios con ciertos tipos de discapacidad.
    • WCAG anima a cumplir técnicas de niveles de cumplimiento superiores

    Técnicas de asesoramiento:

    • Pueden aumentar la accesibilidad
    • Van más allá de los requisitos establecidos por WCAG 2.0
    • No son calificadas como técnicas suficientes debido a que no son suficientes para cumplir con todos los requisitos de los Criterios de Conformidad, no son verificables y/o porque son técnicas buenas y efectivas en algunas circunstancias pero no son útiles en otros casos.
    • Estas dos técnicas (suficientes y de asesoramiento) proporcionan una guía para lograr que el contenido sea más accesible
