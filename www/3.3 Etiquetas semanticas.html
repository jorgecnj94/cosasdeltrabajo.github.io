3.3 Etiquetas semánticas de HTML5

Procedamos a ver las etiquetas semánticas de HTML5.
Lo primero,es tener actualizado el ordenador, ya que HTML5 es una nueva version de HTML, y muchos navegadores no soportan estos cambios.


Definición

Una etiqueta se califica como semántica si nos informa sobre lo que va a tratar su contenido.
Las etiquetas semánticas ayudan a definir la estructura del documento, y permiten una mejor indexacion de las paginas. Un ejemplo es la etiqueta <SECTION>,
la cual nos dice QUÉ contiene una sección o capítulo dentro de la página.

Frente a las etiquetas semánticas tenemos otros tipos de etiquetas, como las que afectan al formato (<UL>), las que afectan a multimedia (como <VIDEO>), etc.

El uso de las etiquetas semánticas es bastante simple e intuitivo, y se pueden colocar dentro del texto como se desee, sin existir reglas sobre su colocación. Pero tampoco podemos olvidar que
son utilizadas para mejorar la indexación de la página, por lo que debemos colocarlas con lógica.

También hay que aclarar que las etiquetas semánticas no añaden ningún formato, ni cambian el tipo de letra, ya que podríamos pensar que <HEADER> nos cambiaría el tamaño de letra.
El formato lo definiremos posteriormente con CSS.

Antes de la creación de estas nuevas etiquetas, se utilizaban las etiquetas DIV, las cuales no aportaban información al tema, salvo que se utilizasen con propiedades ID o CLASS, como por ejemplo:   <DIV CLASS = "Cabecera">


Tipos de etiquetas

Las que vamos a estudiar más a fondo son: <HEADER>, <NAV>, <SECTION>, <ARTICLE>, <ASIDE>, <FOOTER>, <FIGURE> y <FIGCAPTION>

El esquema de una página con etiquetas semánticas podría ser el siguiente.

<HEADER>

Se utiliza como una introducción del elemento que la contiene. El HEADER suele llevar el título, la descripción corta y el logo de la página.
Si colocamos un HEADER dentro del BODY, será una introducción de toda la página. Si colocamos un HEADER dentro de un SECTION, contendrá una introducción a esa sección.

El HEADER no es equivalente a la parte superior de la página, aunque frecuentemente está en esa posición.
El HEADER por si mismo no separa contenidos, por lo que no se debe utilizar aislado, sino dentro de otra etiqueta.
Es bastante frecuente utilizar las etiquetas H1, H2, ... para escribir los títulos dentro del HEADER.

<SECTION>

Una SECTION agrupa contenido con un tema común; como por ejemplo, las diferentes partes de un periódico: política, sociedad, deportes, ...
Cada SECTION puede estar encabezada por una etiqueta H1, H2,... dentro de una SECTION suelen haber etiquetas ARTICLE, DIV o P.

Por ejemplo, un artículo de prensa sobre futbol dentro de la sección de deportes.
Una SECTION no debe ser usada con el único fin de dar formato, para eso se utiliza la etiqueta <DIV>.

<ARTICLE>

Representa una unidad de contenido, es decir, contenido que responde a un tema concreto.
El ejemplo más claro es un artículo dentro de una revista, o también, si nos situamos en un foro, un ARTICLE sería un post.

Se pueden anidar un ARTICLE dentro de otro ARTICLE. Dentro de un ARTICLE, el contenido se suele estructurar con las etiquetas <DIV> y <P>.

<NAV>

Esta etiqueta está pensada para contener la barra de navegación, o los enlaces a páginas de ese sitio web.

Se suele colocar en la parte superior de la página, o en un lateral.
Suele estar formada por una lista de enlaces con las etiquetas <UL> y <LI>.

<ASIDE>

Esta etiqueta está diseñada para contenido suplementario, es decir, menos importante que el cuerpo de la página, ya que se coloca en un lateral de ésta.

Esta etiqueta puede contener un tema relacionado indirectamente con el tema principal, como una reseña de un autor; o puede contener un tema no relacionado, como en el caso de publicidad.

<FOOTER>

Normalmente, esta etiqueta se coloca al final de la página, y contiene información del tipo: autor, copyright, contacto, mapa del sitio, etc.

Recapitulando, mostremos un ejemplo de todo lo visto hasta ahora:

<body>

   <header>
   <h1>Recetas</h1>
      <nav>
         <a href="#">Carnes</a>
         <a href="#">Pescados</a>
      </nav>
   </header>

   <section>
   <h2>Recetas de carne</h2>
      <article>
         <h3>Conejo al ajillo</h3>
         <p>Explicación de la receta 1</p>
      </article>
      <article>
         <h3>Choto a la brasa</h3>
         <p>Explicación de la receta 2</p>
      </article>
   </section>

   <section>
   <h2>Recetas de pescado</h2>
      <article>
         <h3>Dorada a la sal</h3>
         <p>Explicación de la receta 3</p>
      </article>
   </section>

   <aside>
      <p>Publicidad de vodafone</p>
   </aside>

   <footer>
      <p>Autores: los ScrumMasters</p>
   </footer>
</body>

Otras etiquetas...

Aquí podemos resaltar unas nuevas etiquetas semánticas, como:

la etiqueta <FIGURE> usada para imágenes y gráficos,
la etiqueta <FIGCAPTION> que es un pie de imagen,
la etiqueta <TIME> para definir fechas y horas,
y las etiquetas <MAIN>, <MARK>, <SUMARY> Y <DETAILS>.

Veamos un ejemplo con el código con el que vamos a usar estas etiquetas:

<figure>
   <img src="img/IES.jpg" alt="IES Virgen del Carmen" width="500" height="300">
   <figcaption>Fig.1 - Entrada del instituto Virgen del Carmen de Jaén.</figcaption>
</figure>
