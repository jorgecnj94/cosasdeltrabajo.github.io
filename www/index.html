Tamaño de fuentes en HTML

El tamaño de las fuentes se puede especificar de dos maneras, una de ellas por medio de un número del 1 al 7 (de más pequeño a más grande) del siguiente modo
<font size=1> Esta es la letra más pequeña que se puede conseguir </font>
que se verá como
Esta es la letra más pequeña que se puede conseguir

Otra forma es por medio de una referencia relativa:
 <font size="+1"> Esto es igual que poner size=4 </font>
que se ve como
 Esto es igual que poner size=4

El tamaño por defecto es 3

Párrafos y bloques en HTML

Para definir y separar bloques de texto se emplean una serie de etiquetas que definen los párrafos, texto preformateado o bloques con significado especial como direcciones o citas.
Etiquetas de bloques:

<P> -Se utiliza para que los párrafos queden separados por una línea en blanco. Si solo quieres escribir un punto y aparte debes usar <BR>. Las marcas inicial y final son <P> y </P>. La etiqueta <P> admite los atributos:

ALIGN="left" , ALIGN="right" , ALIGN="center" y ALIGN="justify" para alinearlo a la izquierda, a la derecha, centrarlo o justificarlo totalmente. Por defecto está alineado a la izquierda.

<PRE> -El texto insertado entre las marcas <PRE> y </PRE> será visualizado por el navegador respetando el formato con el que fue escrito en el fichero fuente HTML. Esta etiqueta es muy útil para escribir más de un espacio o para hacer pequeñas tablas

<ADDRESS> -Empleada para indicar que un texto representa una dirección o una firma. Generalmente se presenta en cursiva y tabulado.

<BLOCKQUOTE> -Se representa con tabulaciones a la izquierda y derecha. Toda los párrafos de esta página están entre las etiquetas <BLOCKQUOTE> y </BLOCKQUOTE>, de ese modo se consigue que el texto se presente con márgenes a ambos lados.

<BR> -Este elemento sólo tiene marca inicial e indica un salto de línea, es decir un punto y aparte sin separar el párrafo. Esta etiqueta no tiene su correspondiente de cierre.

<HR> -Se emplea para representar una línea horizontal. Tampoco tiene pareja de cierre. Se pueden emplear los atributos:

ALIGN="left" , ALIGN="right" , para alinearla a la izquierda o a la derecha. Por defecto aparece centrada

WIDTH="66%" SIZE="3", para especificar el ancho en % y el alto en píxels

COLOR="#0000FF" , para especificar el color

 

3.3 Etiquetas semánticas de HTML5

Procedamos a ver las etiquetas semánticas de HTML5.
Lo primero,es tener actualizado el ordenador, ya que HTML5 es una nueva version de HTML, y muchos navegadores no soportan estos cambios.


Definición

Una etiqueta se califica como semántica si nos informa sobre lo que va a tratar su contenido.
Las etiquetas semánticas ayudan a definir la estructura del documento, y permiten una mejor indexacion de las paginas. Un ejemplo es la etiqueta <SECTION>,
la cual nos dice QUÉ contiene una sección o capítulo dentro de la página.

Frente a las etiquetas semánticas tenemos otros tipos de etiquetas, como las que afectan al formato (<UL>), las que afectan a multimedia (como <VIDEO>), etc.

El uso de las etiquetas semánticas es bastante simple e intuitivo, y se pueden colocar dentro del texto como se desee, sin existir reglas sobre su colocación. Pero tampoco podemos olvidar que
son utilizadas para mejorar la indexación de la página, por lo que debemos colocarlas con lógica.

También hay que aclarar que las etiquetas semánticas no añaden ningún formato, ni cambian el tipo de letra, ya que podríamos pensar que <HEADER> nos cambiaría el tamaño de letra.
El formato lo definiremos posteriormente con CSS.

Antes de la creación de estas nuevas etiquetas, se utilizaban las etiquetas DIV, las cuales no aportaban información al tema, salvo que se utilizasen con propiedades ID o CLASS, como por ejemplo:   <DIV CLASS = "Cabecera">


Tipos de etiquetas

Las que vamos a estudiar más a fondo son: <HEADER>, <NAV>, <SECTION>, <ARTICLE>, <ASIDE>, <FOOTER>, <FIGURE> y <FIGCAPTION>

El esquema de una página con etiquetas semánticas podría ser el siguiente.

<HEADER>

Se utiliza como una introducción del elemento que la contiene. El HEADER suele llevar el título, la descripción corta y el logo de la página.
Si colocamos un HEADER dentro del BODY, será una introducción de toda la página. Si colocamos un HEADER dentro de un SECTION, contendrá una introducción a esa sección.

El HEADER no es equivalente a la parte superior de la página, aunque frecuentemente está en esa posición.
El HEADER por si mismo no separa contenidos, por lo que no se debe utilizar aislado, sino dentro de otra etiqueta.
Es bastante frecuente utilizar las etiquetas H1, H2, ... para escribir los títulos dentro del HEADER.

<SECTION>

Una SECTION agrupa contenido con un tema común; como por ejemplo, las diferentes partes de un periódico: política, sociedad, deportes, ...
Cada SECTION puede estar encabezada por una etiqueta H1, H2,... dentro de una SECTION suelen haber etiquetas ARTICLE, DIV o P.

Por ejemplo, un artículo de prensa sobre futbol dentro de la sección de deportes.
Una SECTION no debe ser usada con el único fin de dar formato, para eso se utiliza la etiqueta <DIV>.

<ARTICLE>

Representa una unidad de contenido, es decir, contenido que responde a un tema concreto.
El ejemplo más claro es un artículo dentro de una revista, o también, si nos situamos en un foro, un ARTICLE sería un post.

Se pueden anidar un ARTICLE dentro de otro ARTICLE. Dentro de un ARTICLE, el contenido se suele estructurar con las etiquetas <DIV> y <P>.

<NAV>

Esta etiqueta está pensada para contener la barra de navegación, o los enlaces a páginas de ese sitio web.

Se suele colocar en la parte superior de la página, o en un lateral.
Suele estar formada por una lista de enlaces con las etiquetas <UL> y <LI>.

<ASIDE>

Esta etiqueta está diseñada para contenido suplementario, es decir, menos importante que el cuerpo de la página, ya que se coloca en un lateral de ésta.

Esta etiqueta puede contener un tema relacionado indirectamente con el tema principal, como una reseña de un autor; o puede contener un tema no relacionado, como en el caso de publicidad.

<FOOTER>

Normalmente, esta etiqueta se coloca al final de la página, y contiene información del tipo: autor, copyright, contacto, mapa del sitio, etc.

3.5 Tablas en HTML5

Tablas

Las tablas son elementos fundamentales del HTML, ya que mejora la puesta visual de la página. Son conjuntos de filas y columnas que muestran el texto y las imágenes que nosotros queramos. Las tablas admiten todo tipo de formatos y por tanto nos permite diseñar disposiciones complejas en las páginas.

Tablas simples

Para realizar una tabla simple utilizaremos la etiqueta table , con la que indicaremos cada fila con tr para realizar una fila y td para realizar una columna.
Con el atributo border podemos modificar el ancho del borde de la tabla.

3-6 MODELO DE CAJAS:

La siguiente imagen representa el modelo de cajas:

Cada elemento se representa como una caja rectangular, en la que tienen los siguientes elementos:

1- Width y Height

Son el ancho y el alto de la caja. Ejemplo:

         div {             width: 320px;             height: 320px;         }

2- Padding

Es el area alrededor del contenido. Es transparente. Ejemplo:

         div {             padding: 25px;         }

3- Border

Es el borde entre el padding y el contenido. Suele tener 3 valores, el primero el ancho del borde (usualmente en pixeles, pero puede tener otra unidad), el segundo el estilo (puede tener puntitos, rallas, guiones...) y el tercero el color. Ejemplo:

         div {             border: 25px solid green;         }

4- Margin

Es el margen exterior. Es transparente. Ejemplo:

         div {             margin: 25px;         }

3.8 Enlaces

Lo que caracteriza a la web como medio de comunicación es la navegación por medio de hipervínculos (enlaces).
Esta conexión entre archivos permite al usuario elegir el sentido de la navegación.


En HTML5, los enlaces se marcan con la etiqueta <a> </a> y el atributo principal es href="" , donde se escribe la
ubicación del archivo de destino, el cual puede estar en la misma carpeta que el archivo que lo está llamando, en otra carpeta, o en un sitio web.

Dentro de las etiquetas <a href=""></a> se puede colocar cualquier elemento html, el cual funcionará como botón (generalmente se suele colocar un texto o una imagen).
Si es un texto, el navegador por defecto lo muestra en color azul y subrayado.
Si es una imagen, algunos navegadores le suelen agregar un borde azul.

Tipos de enlaces

Existen diferentes tipos de enlaces que detallamos a continuación:

Enlaces externos

Son los enlaces entre diferentes sitios web. Si queremos vincular un sitio web con otro, tenemos que indicar la ruta absoluta del sitio escribiendo la url completa.
La ruta es el valor que se escribe dentro del atributo href="", y se puede clasificar en rutas absolutas o relativas.

Las rutas absolutas, como ya hemos visto, son las que definen la ubicación completa de un archivo en la web, y empiezan desde http://nombredelsitio.com/.../archivo.extensión
Las rutas relativas, en cambio, definen la ubicación del archivo de destino en relación a la ubicación del archivo que lo está llamando.

Pongamos unos ejemplos:

Ruta absoluta:
<a href="https://www.https://www.iesvirgendelcarmen.com/moodle/course/index.php?categoryid=23">Moodle Virgen del Carmen</a>

La cual genera este enlace: Moodle Virgen del Carmen

En algunos casos, se suele usar el atributo target="" , que nos permite abrir el enlace en una pestaña nueva. Por ejemplo,

<a href="http://google.com" target="_blank>Google en una nueva pestaña</a>

Genera este enlace: Google en una nueva pestaña


Enlaces internos entre archivos .html

Son los enlaces principales de un sitio. Si en el archivo index.html, queremos agregar un vínculo a Página de prueba.html, el código sería así:

Ruta relativa:
<a href="../extra/Página de prueba.html">Página de prueba</a>

La cual genera este enlace: Página de prueba


Enlaces internos: anclas

Las anclas son enlaces hacia un punto determinado dentro de un html. Se usa en varios casos, por ejemplo en una
página de preguntas frecuentes, donde las respuestas están desarrolladas debajo de las preguntas (foros con posts de respuestas).
También en los textos largos, al finalizar muchas veces se coloca un botón para subir, lo cual es un ancla.

El punto de destino tiene que estar marcado con el atributo id="algo" , y en el enlace se coloca un numeral (#) seguido del nombre. Por ejemplo:

Inicio de la página
con código:     <a href="#Inicio">Inicio de la página</a><br>

"salta" a este párrafo colocado en el inicio de esta página:

<p id="Inicio">Inicio de la página</p></p>


Enlaces a una dirección de correo

Se puede vincular una dirección de correo para que abra el programa de correo predeterminado. El código sería:

Consúltenos a <a href="mailto:info@iesvirgendelcarmen.com">info@iesvirgendelcarmen.com</a>

Que produce como resultado:   Consúltenos a info@iesvirgendelcarmen.com


Se le puede añadir más contenido al mensaje, añadiendo '?' al final del correo, y separando los siguientes valores con '&'.
Algunos valores que podemos añadir son: 'subject', 'body', 'cc' para copia, 'bcc' para copia oculta...
Veámoslo con el mismo ejemplo de antes:

Consúltenos a <a href="mailto:info@iesvirgendelcarmen.com?subject=Duda&body=Buenos%20Días,%20">info@iesvirgendelcarmen.com</a>

Que produce como resultado:   Consúltenos a info@iesvirgendelcarmen.com

En nuestro ordenador, se nos abriría la ventana del correo electrónico tal que así:

Este recurso está bastante cuestionado, debido a dos razones:
Primero porque muchos usuarios utilizan correo webmail (Gmail, Yahoo, Hotmail) por lo que este enlace no les sirve;
Y segundo porque estamos facilitando a robots que encuentren nuestro correo y lo incluyan en una base de datos para spam.


Enlaces para ampliar una imagen

Es muy común tener en nuestra página web una galería con miniaturas de fotos que vinculan a imágenes ampliadas.
Si combinamos este recurso con código javascript, logramos galerías de fotos muy atractivas. El código HTML sería así:

<a href="img/IES.jpg"><img src="img/IES-miniatura.jpg" alt="Entrada del IES Virgen del Carmen"></a>


Que produciría como salida la imagen en miniatura:    

Donde podemos hacer click y se nos mostrará la imagen ampliada.


Enlaces para descargar archivos

Esto es mas de lo mismo ya visto en los anteriores apartados, puesto que dentro del atributo href="" procedemos a colocar la ruta del archivo.
Puesto que el navegador no puede abrir en su interior archivos tales como .zip o .rar, éste los descarga en el ordenador del usuario.
Veámoslo con un ejemplo:

<a href="img/img.zip">Descarga todas las fotos vistas en esta sección del manual</a>


Generará el enlace siguiente:    Descarga todas las fotos vistas en esta sección del manual
